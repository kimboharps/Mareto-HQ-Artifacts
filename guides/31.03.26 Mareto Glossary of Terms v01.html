Glossary of Terms
Key Terminology for Working in Mareto

About This Glossary

This glossary defines the key terms and concepts you will encounter when using Mareto. It is designed to help your team develop a shared vocabulary so that data is entered consistently across programs, and everyone speaks the same language when working within the platform.

How to Use This Guide

Terms are organized alphabetically. Core structural terms (Person, Case, Event) form the foundation of how data is organized in Mareto. All other terms build on that foundation. If you are new to Mareto, start with those three entries first.

A
TermDefinition
AssessmentA structured evaluation recorded within a case file. Assessments capture a client's needs, risks, strengths, or progress at a specific point in time. Organizations may use standardized assessment tools or custom forms built during implementation.
AttendanceA record of a client's participation in a scheduled session, group program, or appointment. Attendance is logged as an event and linked to both the person record and the relevant case or group session.
AutomationA rule-based action that the system performs automatically when a trigger condition is met. Examples include follow-up reminders, consent expiry alerts, email notifications for payment approvals, and discharge workflows that cancel future appointments.
B
TermDefinition
Bed LogAn event type used by shelter and residential programs to track nightly occupancy. Bed logs record which units or beds are occupied, by whom, and for how long.
Billable HoursTime logged against a case or contract for billing or accountability purposes. Tracks attendance type, location, and duration for staff working on funded programs.
C
TermDefinition
CaseA container for a person's relationship with a specific program or service. A case holds all information that evolves over time: status, goals, assessments, case plans, and outcomes. A single person can have multiple cases across different programs, and multiple cases within the same program.
Case NoteA narrative record documenting an interaction, observation, or update related to a client's case. Case notes are logged as events and are timestamped automatically.
Case PlanA documented plan of action within a case file that outlines the goals, services, and steps agreed upon for a client. Case plans link to goals and assessments within the same case.
Case StatusThe current stage of a case within its lifecycle. Common statuses include Open, Pending, On-Hold, Transferred, and Closed. Statuses are configurable per program.
Check RunA batch process for approved payment requests. A Check Run groups multiple approved payments together, generates a unique ID, aggregates totals, and cascades status changes (Draft, Submitted, Paid) across all connected payment requests and purchase orders.
Collateral ContactAn external partner or professional (such as a probation officer, child welfare worker, or partner agency contact) linked to a client's case file. Collateral contacts are stored in a shared organizational contact book and are searchable by name and organization.
CommentsAn internal communication channel within Mareto. Comments can be attached to person records or case files and are visible to relevant staff. They enable real-time team coordination without leaving the platform.
ConsentA record attached to a person's profile documenting their authorization for data collection, information sharing, or program participation. Consent records include the date, uploaded signed documents, and expiry dates. Automated reminders notify staff when renewal is due.
D
TermDefinition
DashboardA visual summary of key metrics displayed on the Mareto homepage or within a specific program. Dashboards are configurable per role, so a caseworker sees their caseload while a director sees organization-wide analytics.
Data MigrationThe process of transferring existing records from a previous system (such as spreadsheets, legacy databases, or other software) into Mareto. HelpSeeker's data team handles matching, deduplication, and cleanup during migration.
DiscoveryThe first phase of a Mareto implementation. During discovery, HelpSeeker gathers all relevant documents (intake forms, assessments, consent forms, funder reporting templates) and maps out every program, workflow, and reporting requirement to inform the data model.
Drop-DownA predefined list of options for a data field. Mareto is designed around drop-down menus rather than open text fields wherever possible. This ensures data quality and consistent reporting across the organization.
Duplicate PreventionA built-in feature that uses fuzzy matching to flag potential duplicate person records before they are created. If a similar name already exists in the system, Mareto alerts the user. Existing duplicates can be merged using the file-merging tool.
E
TermDefinition
EventA point-in-time record tied to a person or case. Events are timestamps that build a client's timeline: case notes, attendance, incidents, progress updates, shift logs, and more. Event types are dynamic, meaning the form fields change depending on the type selected (for example, a crisis incident shows different fields than a routine progress note).
Event TypeA category that defines what kind of event is being recorded. Each event type determines which form fields appear. Common event types include case note, attendance, crisis incident, progress note, follow-up, group session, shift log, and bed log.
External Referral FormA URL-based form that allows external partners (such as police, probation officers, or other agencies) to submit referrals into Mareto. Submissions are placed in a holding area and are not automatically created as person records. The system checks for name matches and flags potential duplicates.
F
TermDefinition
Family LinkingThe ability to connect related person records together across programs. For example, a parent accessing the food bank and their child enrolled in youth programming are linked, giving staff a holistic view of the family's engagement across the entire organization.
File MergingThe process of combining two duplicate person records into one. When a duplicate is discovered, file merging consolidates all associated cases, events, and history into a single record.
Filterable ReportA report that can be narrowed by demographics, time period, program, status, or any other captured data element. Filterable reports can be exported as CSV, PDF, or image. A rollup feature supports monthly funder reporting requirements.
Form BuilderA tool within Mareto that generates external links for data collection. Form Builder supports website intake forms, subcontractor reporting, partner data submissions, and client-facing uploads. Data submitted through these forms flows directly into Mareto and triggers staff notifications.
Fuzzy MatchingAn algorithm that identifies potential duplicate records by comparing name similarity. For example, if "Robert Smith" already exists and someone enters "Bob Smith," the system flags the match for review.
G
TermDefinition
Goal TrackingA feature that allows staff to define, monitor, and measure client goals within a case. Goals link to assessments and services, enabling progress tracking over time and across programs.
Group SessionA recurring program activity (such as a workshop, support group, or class) that multiple clients attend. Group sessions track active and inactive status, and attendance is recorded per session through the event log.
I
TermDefinition
ImplementationThe process of configuring Mareto for your organization. Implementation includes discovery, data model design, form and workflow building, role-based access setup, training, and go-live support. Every implementation is custom-built to match your programs and processes.
O
TermDefinition
OccupancyA live count of available and occupied rooms, beds, or units within a shelter or residential program. Occupancy updates automatically as case files open and close, and tracks nights in service and bed types.
OntologyThe structured framework that defines how data is organized across the entire platform. In Mareto, a person is always a person, a case is always a case, and a service is always a service, regardless of which program uses them. This consistency is what allows data to roll up meaningfully from the caseworker level to the executive level.
Outcome TrackingThe measurement of client progress against defined goals over time. Mareto supports outcome measures at multiple points, aggregated over time and visualized as progress reports. Automated follow-up surveys can be scheduled (for example, six months post-referral).
P
TermDefinition
Payment EntitlementA payment request tied to a case. Payment entitlements include vendor selection, general ledger codes, fund allocation, and automated email notifications to approvers. Supports recurring payments.
PersonThe foundational record in Mareto. A person is anyone who interacts with the organization: a client, participant, community member, family member, volunteer, or staff member. There is only one person record per individual, even if they are enrolled in multiple programs simultaneously. This prevents duplication and enables a holistic view across the organization.
Person RegistryThe centralized directory of all person records in the system. The Person Registry stores demographics, contact information, family connections, emergency contacts, and person type designations. It serves as the single source of truth for who the organization serves.
Pre-Client ContactA record of communication (phone calls, emails, inquiries) with an individual before they are formally registered as a client. Pre-client contacts link to the person's profile when they are later onboarded into a program.
ProgramA distinct service area or operational stream within your organization (such as housing, youth services, counselling, or food distribution). Each program in Mareto has its own case types, event types, forms, dashboards, and reporting structures.
R
TermDefinition
ReferralA formal request to connect a client with a service, either internally (between programs within the same organization) or externally (to a partner agency). Internal referrals trigger notifications and emails to receiving staff. External referrals can share non-confidential information with partner organizations. Referral outcomes (successful or unsuccessful) and volume by source are tracked.
Restricted NoteA clinical case note that is locked so that only the clinician who entered it can view it. Restricted notes operate on a separate access layer from standard role-based controls. Even managers and directors cannot access them without explicit permission.
Role-Based AccessA permissions system that controls what each user can see and do within Mareto. Access is scoped to specific programs, case files, and individual fields. A caseworker sees only their assigned clients, a program manager sees their entire program, and a director sees organizational dashboards. Some organizations have thirty or more distinct roles configured.
Rollup ReportA reporting feature that aggregates data across multiple cases, programs, or time periods into a single summary. Commonly used for monthly funder submissions where data from individual case records must be combined into program-level or organization-level totals.
S
TermDefinition
SandboxA limited trial environment set up with a defined feature set (basic person, case, and event tracking, plus appointments). A sandbox allows staff to experience the system for a few weeks before committing to a full implementation.
SchedulingA centralized calendar feature for booking one-on-one sessions, group programs, and housing intakes. Appointments attach to person and case records. Calendar views are configurable per role.
Soft ArchivingThe process of temporarily closing a case file with the ability to reactivate it later. Each program can have different non-disclosure and retention dates. Closing a case in one program does not affect active cases in other programs for the same person.
Survey ToolA built-in feature for creating and distributing surveys within Mareto. Responses are stored alongside case data. Supports automated follow-up surveys (for example, at six months post-referral) and multiple outcome measures aggregated over time.
T
TermDefinition
TaxonomyThe global set of categories, labels, and drop-down values used across all programs in your Mareto instance. During implementation, HelpSeeker harmonizes every category across your organization so that data rolls up consistently from program-level operations to organization-level reporting.
Two-Factor Authentication (2FA)A security measure requiring all users to verify their identity via an email-based code on every login, in addition to their password.
U
TermDefinition
Unnamed ParticipantA count-based attendance record used for group sessions where individual names are not collected. The record captures date, class name, location, and total attendance. For mixed groups, named clients are recorded alongside an unnamed participant count.
W
TermDefinition
WorkflowA defined sequence of steps or actions within Mareto that guides how data moves through a process. Workflows can include automated triggers (such as sending notifications when a referral is received) and status changes that cascade across related records.
